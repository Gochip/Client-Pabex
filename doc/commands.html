Comandos del cliente

1- Desconectar.

  • DISCONNECT
  • Ninguno
  • Ninguna
  • msg_info: Es un mensaje indicando que el cliente se desconectó.
  • No genera ningún error.
  • No genera ninguna advertencia.
  • DisconnectResponse: A los demás integrantes del grupo.

2- Comando crear grupo.

  • CREATE_GROUP 'nombre del grupo' <-n 'natural'> <-p 'cadena'> <-f 'yes|no'>
  • Necesita un único valor obligatorio y es el nombre del grupo.
  • n: Es la cantidad de jugadores máxima que acepta el grupo.
  • p: Es la clave del grupo.
  • f: Determina si el grupo se borra cuando el administrador se desconecta. Un valor de 'yes' indica que el grupo se borrará cuando el administrador se desconecte. Un valor 'no' indica que el grupo no se borrará cuando el administrador se desconecte y pasará la administración a cualquiera de los demás clientes conectados. Si el administrador es el único integrante en el grupo entonces se borrará el grupo independientemente de la opción f.
  • id_group: Es el id del grupo creado.
  • No puede generar ningún error.
  • No puede generar ninguna advertencia.
  • No puede generar ninguna notificación.

3- Unirse a un grupo.

  • JOIN 'id del grupo' <-p 'cadena'>
  • Necesita el id del grupo al cual se va a unir.
  • p: Es la clave del grupo.
  • id_group: Es el id del grupo al cual se acaba de unir.
  • id_admin: Es el id del administrador del grupo al cual se acaba de unir.
  • (1) NOT_EXISTS_GROUP: No existe el grupo especificado por el id.
  • (3) GROUP_IS_FULL: El grupo está lleno.
  • (4) DUPLICATE_IN_GROUP: El cliente ya está conectado a este grupo.
  • (5) NOT_PASSWORD: El cliente no proveyó una clave a un grupo privado.
  • (6) INVALID_KEY: Clave inválida.
  • Ninguna
  • InformationResponse: A los demás integrantes del grupo.
    • Source: join

4- Abandonar un grupo.

Sintaxis: LEAVE_GROUP 'id del grupo'

Valor obligatorio: Necesita el id del grupo al cual se va a unir.

Opciones: p: Es la clave del grupo.

Respuesta: msg_info: Es un mensaje de información.

Errores: NOT_EXISTS_GROUP: No existe el grupo especificado en el id. NOT_EXISTS_CLIENT_IN_GROUP: El cliente no existe en el grupo especificado en el id.

Advertencias: No devuelve ninguna advertencia.

Mensaje indirectos: InformationResponse (clave:valor) source: 'leave_group' id_group: Id del grupo abandonado. deleted_group: Indica si el grupo ha sido eliminado. (Se puede usar un Interpreter para evitar usar estas claves).

Función interna: El cliente que ejecuta este comando abandona el grupo. Si dicho cliente no es el administrador del grupo entonces simplemente se lo elimina al cliente del grupo. En cambio, si el cliente es el administrador del grupo entonces se evalúa el atributo forceToDelete. Si este atributo es true entonces se procede a borrar el grupo informando a todos los integrantes del grupo lo ocurrido. Por el contrario, si este atributo es false, se procede a entregarle la administración del grupo a otro cliente.

9- Agregar atributo.

Sintaxis: ADD_ATTRIBUTE 'key1, key2,..., key3' 'value1, value2,..., value3' <-v 'true|false'>

Valor obligatorio: key1, key2,..., key3: Son las claves de los atributos. value1, value2,..., value3: Son los valores de los atributos.

Opciones: v: (Verbose) Indica si debe avisar a los clientes de los grupos que forma parte.

Respuesta: msg_info: Es un mensaje indicando que el cliente agrego atributo/s.

Errores: INCORRECT_LENGTH_KEY_VALUE: La cantidad de claves difiere a la cantidad de valores.

Advertencias: Ninguna

Mensaje indirectos: Envía un Informatio Response de tipo 'add_attribute' a todos los clientes que están en algunos de sus grupos.
